# Frontend Acceptance Checklist

更新时间：2026-05-24

本清单按个人 vibecoding 项目的验收标准记录，不追求工业级自动化覆盖，只确认 `0.2.0` 的核心使用路径已经可用。

## 桌面端

- [x] 默认进入页面后展示赛程视图。
- [x] 可以切换赛程 / 赛果视图。
- [x] 可以使用今天、明天、未来 7 天、最近 7 天快捷项。
- [x] 可以手动选择开始日期和结束日期，范围被限制在 31 天内。
- [x] 可以按游戏、状态、赛事、队伍、赛区/国家、阶段和关键词筛选。
- [x] 可以按时间、状态、更新时间和赛事名称排序。
- [x] 比赛卡片展示时间、游戏、赛事、队伍、状态、BO、比分和胜者。
- [x] 可以展开详情查看来源、更新时间、总比分和小局摘要。
- [x] 刷新页面后 URL 中的筛选条件可以恢复。

## 移动端

- [x] 筛选控件在窄屏下纵向排列，可继续操作。
- [x] 比赛卡片在窄屏下单列展示，队伍比分不挤出屏幕。
- [x] 详情展开后内容可阅读，不依赖横向滚动。

## 状态

- [x] loading 状态有明确展示。
- [x] empty 状态会根据赛程 / 赛果给出不同文案。
- [x] error 状态支持重试。
- [x] stale 数据会显示提示。
- [x] 开发环境缺少 PandaScore token 时可回退到 mock 数据。

## 本次验证命令

```bash
npm test
npm run build
```

命令通过后，`0.2.0` 可按个人项目标准视为功能完成。
