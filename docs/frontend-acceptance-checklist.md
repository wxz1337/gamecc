# Frontend Acceptance Checklist

更新时间：2026-05-25

本清单按个人 vibecoding 项目的验收标准记录，不追求工业级自动化覆盖，只确认 `0.4.0` 时间线赛程浏览后的核心使用路径已经可用。

## 桌面端

- [x] 默认进入页面后展示赛程视图。
- [x] 首页展示个人观赛面板，包括当前日期、已载入比赛数和进行中比赛数。
- [x] 可以使用自然周日期条、前一天/后一天和日期选择器切换日期。
- [x] 可以按游戏、赛区、赛事级别和状态筛选。
- [x] 状态筛选包含进行中&未开始、已结束两类。
- [x] 进行中&未开始只展示这两类比赛，已结束只展示已结束比赛，两者都按时间正序排列。
- [x] 选择今天以前的日期会自动切到已结束，选择今天或未来日期会自动切到进行中&未开始。
- [x] 游戏、赛区、赛事级别和状态 Tabs 对齐且可横向滚动。
- [x] 默认定位当天进行中、即将开始或最近结束的比赛。
- [x] 选择日期后不能向上加载选中日前的赛程，只能向下加载到本周最后一天。
- [x] 时间线限制在选中日至本周最后一天内，超出范围需通过日期栏选择对应日期查看。
- [x] 列表底部会显示已加载到哪一天、是否正在加载更多、是否已到本周最后一天。
- [x] 今日/当日重点赛事区域会跟随当前可见日期展示进行中或高等级赛事。
- [x] 比赛卡片展示时间、游戏、赛事、队伍、状态、BO、比分和胜者。
- [x] 可以展开详情查看来源、更新时间、总比分和小局摘要。
- [x] 刷新页面后 URL 中的筛选条件可以恢复。
- [x] LoL 和 VALORANT 详情中的直播链接指向个人常用 B 站直播间。

## 移动端

- [x] 筛选控件在窄屏下纵向排列，可继续操作。
- [x] 日期条在窄屏下保持可读，不挤出屏幕。
- [x] 比赛卡片在窄屏下单列展示，队伍比分不挤出屏幕。
- [x] 详情展开后内容可阅读，不依赖横向滚动。

## 状态

- [x] loading 状态有明确展示。
- [x] empty 状态会根据赛程 / 赛果给出不同文案。
- [x] error 状态支持重试。
- [x] stale 数据会显示提示。
- [x] 开发环境缺少 PandaScore token 时可回退到 mock 数据。

## UI / 动效

- [x] 使用 Tailwind CSS 统一样式。
- [x] 使用 shadcn/ui 风格基础组件承载按钮、卡片、标签和骨架屏。
- [x] 使用 lucide-react 图标提升状态和操作识别。
- [x] 使用 framer-motion 提供轻量动画，不过度干扰阅读。
- [x] 卡片 hover、Tab 切换、列表出现和详情展开都有轻量反馈。

## 本次验证命令

```bash
npm run typecheck
npm test
npm run build
```

命令通过后，`0.4.0` 可按个人项目标准视为功能完成。
