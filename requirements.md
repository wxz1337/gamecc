# 电竞赛程聚合网站需求文档

## 1. 项目背景

我平时会观看多种电竞比赛，包括 Counter-Strike 2、VALORANT 和英雄联盟。当前需要每天分别打开不同网站查询当天赛程，再手动对照比赛时间安排观赛计划，过程重复且低效。

本项目希望做一个个人使用的电竞赛程聚合网站，将多个项目的当日赛程统一展示，并按照比赛开始时间排序，帮助快速决定当天看哪些比赛。

## 2. 项目目标

### 2.1 核心目标

- 聚合展示 CS2、VALORANT、英雄联盟的赛程。
- 默认展示当天赛程。
- 所有比赛按开始时间升序排列。
- 时间统一转换为北京时间展示。
- 支持按游戏项目筛选。
- 尽量减少手动查网站的时间。

### 2.2 非目标

第一版不追求以下能力：

- 不做公开运营网站。
- 不做用户注册、登录和多用户系统。
- 不做复杂社区功能。
- 不做实时比赛数据面板，例如击杀、经济、回合详情、英雄数据。
- 不做博彩、赔率或预测功能。
- 不爬取非官方页面作为主要数据来源。

## 3. 目标用户

### 3.1 用户画像

主要用户是我自己：

- 同时关注多个电竞项目。
- 每天需要快速知道今天有哪些值得看的比赛。
- 更关心赛程、时间、赛事名和参赛队伍，而不是复杂数据统计。
- 使用场景主要是电脑浏览器，也可能在手机上查看。

### 3.2 使用场景

- 晚上打开网站，查看今天接下来还有哪些比赛。
- 午休或下班前查看今晚的 CS2、VALORANT、LoL 赛程。
- 查看明天是否有重要比赛，提前安排时间。
- 快速筛选只看某一个游戏项目。

## 4. 数据来源

### 4.1 首选数据源

第一版使用 PandaScore API 作为主要数据源。

理由：

- 覆盖 Counter-Strike、VALORANT、League of Legends 等项目。
- 免费额度对个人使用基本足够。
- 数据结构相对统一，适合做聚合展示。
- 有公开 API 文档，维护成本低于页面爬虫。

### 4.2 数据获取策略

- 后端从 PandaScore 拉取赛程数据。
- 所有时间以 UTC 存储或处理。
- 前端展示时转换为北京时间。
- 后端可以缓存当天、明天和未来几天的赛程，减少 API 调用。
- 个人使用阶段可以先采用简单内存缓存或本地文件缓存。

### 4.3 数据兜底

如果 PandaScore 某些小赛事缺失，第一版可以接受，不额外接入其他数据源。

后续如有需要，再考虑接入：

- Liquipedia API
- 官方赛事站 API
- 手动维护关注赛事列表

## 5. 功能需求

### 5.1 今日赛程列表

作为用户，我希望打开网站后立即看到今天的全部比赛，方便安排观赛时间。

要求：

- 默认展示当天 00:00 至 23:59 的比赛。
- 日期基准使用北京时间。
- 比赛按开始时间升序排列。
- 每场比赛至少展示：
  - 游戏项目
  - 比赛开始时间
  - 赛事名称
  - 参赛队伍
  - 比赛状态
  - BO 赛制，如果数据源提供

### 5.2 游戏项目筛选

作为用户，我希望可以只查看某个游戏的赛程。

要求：

- 支持全部、CS2、VALORANT、英雄联盟四种筛选。
- 筛选后仍然按开始时间排序。
- 筛选状态应在当前页面内保持，刷新后可选做持久化。

### 5.3 日期切换

作为用户，我希望可以查看今天、明天或指定日期的赛程。

MVP 要求：

- 支持今天和明天切换。
- 支持回到今天。

后续增强：

- 支持日期选择器。
- 支持本周视图。

### 5.4 比赛状态展示

作为用户，我希望知道比赛是未开始、进行中还是已结束。

状态可分为：

- 未开始
- 进行中
- 已结束
- 延期或取消，如果数据源提供

### 5.5 空状态

当某天没有比赛时，页面应给出明确提示。

示例：

- 今天暂无已收录赛程。
- 当前筛选条件下没有比赛。

### 5.6 加载和错误状态

页面需要处理数据加载中和 API 请求失败。

要求：

- 加载中显示简洁 loading 状态。
- API 失败时显示错误提示。
- 失败时允许用户手动刷新。

## 6. 界面需求

### 6.1 页面结构

第一版可以采用单页应用结构：

- 顶部区域：站点名称、当前日期、刷新按钮。
- 筛选区域：游戏项目筛选、今天/明天切换。
- 主内容区域：按时间排列的比赛列表。
- 底部区域：数据来源说明。

### 6.2 比赛卡片字段

每场比赛使用一条紧凑卡片或列表项展示。

字段建议：

- 开始时间，例如 `19:00`
- 游戏标签，例如 `LoL`
- 赛事名称，例如 `LPL 2026 Spring`
- 队伍名称，例如 `BLG vs TES`
- BO 信息，例如 `BO3`
- 状态标签，例如 `未开始`

### 6.3 设计风格

- 面向个人效率工具，优先清晰、紧凑、易扫读。
- 不做营销落地页。
- 不需要大幅 hero 区域。
- 适配桌面和手机浏览器。
- 暗色模式可以作为加分项，不是第一版必需。

## 7. 技术方案建议

### 7.1 推荐技术栈

第一版推荐：

- 前端：React + Vite 或 Next.js
- 后端：Next.js API Routes 或 Node.js Express
- 数据源：PandaScore API
- 缓存：内存缓存、本地 JSON 文件或 SQLite
- 部署：本地运行即可，后续可部署到 Vercel、Railway 或 Render

### 7.2 推荐架构

```text
Browser
  |
  v
Frontend UI
  |
  v
Backend API
  |
  v
Cache Layer
  |
  v
PandaScore API
```

### 7.3 API 设计草案

#### 获取赛程

```http
GET /api/matches?date=2026-05-24&game=all
```

请求参数：

- `date`：日期，格式 `YYYY-MM-DD`，按北京时间理解。
- `game`：游戏筛选，可选 `all`、`cs2`、`valorant`、`lol`。

响应示例：

```json
{
  "date": "2026-05-24",
  "timezone": "Asia/Shanghai",
  "matches": [
    {
      "id": "pandascore-match-id",
      "game": "lol",
      "league": "LPL",
      "tournament": "LPL 2026 Spring",
      "beginAt": "2026-05-24T11:00:00Z",
      "displayTime": "19:00",
      "status": "not_started",
      "bestOf": 3,
      "teams": [
        {
          "name": "Team A",
          "acronym": "TA"
        },
        {
          "name": "Team B",
          "acronym": "TB"
        }
      ],
      "streamUrl": null
    }
  ]
}
```

## 8. 数据模型草案

### 8.1 Match

```ts
type GameType = "cs2" | "valorant" | "lol";

type MatchStatus =
  | "not_started"
  | "running"
  | "finished"
  | "postponed"
  | "cancelled";

type Match = {
  id: string;
  game: GameType;
  league: string;
  tournament: string;
  beginAt: string;
  status: MatchStatus;
  bestOf?: number;
  teams: Team[];
  streamUrl?: string | null;
};

type Team = {
  name: string;
  acronym?: string | null;
  imageUrl?: string | null;
};
```

## 9. 缓存策略

### 9.1 MVP 缓存

- 每个日期和游戏组合缓存 10 到 30 分钟。
- 如果缓存存在且未过期，直接返回缓存。
- 如果缓存过期，则重新请求 PandaScore。
- 如果刷新失败且旧缓存存在，可以返回旧缓存并提示数据可能不是最新。

### 9.2 缓存 Key

```text
matches:{date}:{game}
```

示例：

```text
matches:2026-05-24:all
matches:2026-05-24:lol
```

## 10. 环境变量

```env
PANDASCORE_API_TOKEN=your_api_token_here
DEFAULT_TIMEZONE=Asia/Shanghai
```

要求：

- API token 不应提交到 Git。
- 本地使用 `.env.local` 保存。
- 页面不直接暴露 PandaScore token。

## 11. MVP 范围

第一版只做以下内容：

- 接入 PandaScore API。
- 展示今天的 CS2、VALORANT、LoL 赛程。
- 支持今天和明天切换。
- 支持游戏筛选。
- 按北京时间排序。
- 展示加载、错误和空状态。
- 做基础响应式布局。

## 12. 后续扩展

可选增强方向：

- 本周赛程视图。
- 收藏队伍。
- 只显示收藏队伍比赛。
- 开赛前提醒。
- 添加直播平台链接。
- 添加赛事重要程度标记。
- 支持手动隐藏不关注的小赛事。
- 增加桌面通知或 Telegram / 微信通知。
- 增加日历订阅 ICS 导出。
- 接入 Liquipedia 补充缺失赛事。

## 13. 风险和限制

### 13.1 数据完整性

PandaScore 不一定覆盖所有小型赛事。个人使用阶段可以接受，后续根据实际缺失情况再扩展数据源。

### 13.2 API 限额

个人使用的请求量很低，免费额度预计足够。仍应通过缓存降低请求频率。

### 13.3 时间和时区

电竞赛事经常跨时区，必须统一以 UTC 处理，再转换为北京时间展示，避免日期错位。

### 13.4 授权和使用范围

项目定位为个人工具，不做商业运营。若后续公开发布，需要重新确认 PandaScore 的授权条款和使用限制。

## 14. 验收标准

MVP 完成后应满足：

- 打开首页可以看到今天赛程。
- 赛程时间按北京时间显示。
- 比赛按开始时间升序排列。
- 可以筛选 CS2、VALORANT、LoL。
- 可以切换今天和明天。
- 无比赛时有明确空状态。
- API 请求失败时有明确错误提示。
- 页面在桌面和手机宽度下都可正常浏览。
